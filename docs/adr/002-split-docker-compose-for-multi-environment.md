# ADR-002. Разделение Docker Compose на многофайловую структуру

Дата: 2026-07-13

## Проблема

Изначально использовалась схема из двух файлов: `docker-compose.yml` (общая конфигурация) и `docker-compose.override.yml` (локальная разработка).

Такая структура не позволяла внедрить Prometheus и Grafana на Stage-сервер для нагрузочного тестирования, так как они неизбежно попадали бы в Production. Кроме того, автоматический подхват `override.yml` создавал риск случайного применения dev-настроек на серверах.

## Решение

Перейти на явную многофайловую структуру с указанием файлов через флаги `-f`:

- `compose.yaml` — базовая конфигурация (топология, сети, лимиты ресурсов).
- `compose.dev.yaml` — локальная разработка (bind mounts, hot-reload, проброс портов БД).
- `compose.stage.yaml` — Stage-сервер (добавление Prometheus и Grafana для метрик).
- `compose.prod.yaml` — Production (SSL, Certbot, порт 443).

## Заметка

Такой подход обеспечивает полную изоляцию сред и готов к переиспользованию в будущих проектах как универсальный шаблон.
