# ADR-003. Управление rate limiting через переменную окружения

Дата: 2026-07-24

## Проблема

Для нагрузочного тестирования на Stage требуется отключать rate limiting без изменения кода. Жёсткая привязка к заголовку `X-Load-Test` усложняет тесты и не покрывает все сценарии.

## Решение

Добавить настройку `RATE_LIMIT_ENABLED: bool = True` в `Settings`. Передавать флаг `enabled` в `Limiter` slowapi. Переопределять значение через `.env.stage`.

## Заметка

- Dev: `true` (по умолчанию) — тестирование работы rate limiting
- Stage: `false` (переопределение) — нагрузочное тестирование без защиты
- Prod: `true` (по умолчанию) — защита в боевом режиме
