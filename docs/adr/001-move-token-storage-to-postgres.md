# ADR-001. Перенос хранения токенов из Redis в PostgreSQL

Дата: 2026-07-10

## Проблема

Для интеграции с MCP-клиентами требуется поддержка долгоживущих Bearer-токенов. Использование короткоживущих токенов (15 минут) потребовало бы от MCP-клиента автоматически выполнять повторную аутентификацию по логину и паролю, что усложняет взаимодействие.

Хранить долгоживущие токены в Redis нецелесообразно, поскольку они будут теряться при очистке или перезапуске Redis. Дублирование хранения токенов в Redis и PostgreSQL для небольшого проекта является избыточным.

## Решение

Перенести хранение токенов аутентификации из Redis в PostgreSQL.

Токены являются долгоживущими и действуют до их явного отзыва.

## Заметка

Для текущей нагрузки проекта проверка токенов через PostgreSQL является достаточной. Дополнительный слой кэширования не требуется.
